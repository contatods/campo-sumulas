Manual do sistema

O que esse sistema faz

Gera súmulas impressas para eventos de CrossFit. Cada workout vira uma folha A4 (ou um lote de folhas, uma por atleta) pronta pra Ctrl+P → PDF.

Fluxo típico

  1. Configurar nome do evento, datas e logos (botão ⚙ Configurar).
  2. Importar Excel do organizador com workouts, cronograma e atletas. O sistema reconhece 4 formatos automaticamente.
  3. Revisar dias, categorias e workouts no editor. Ajustar se necessário (mover entre dias, editar movimentos, etc).
  4. Gerar súmulas — baixa um ZIP com pastas Dia/Categoria/Workout em HTML.
  5. Imprimir: abre o HTML no browser e Ctrl+P → salvar como PDF ou imprimir.

Tudo é salvo automaticamente

O canto superior direito mostra ✓ salvo HH:MM sempre que o estado é gravado no navegador. Se der F5 ou fechar a aba, ao voltar tá tudo lá. Não precisa de botão "salvar".

Quer um backup externo? Veja a aba Backup.

Formatos de Excel suportados

O parser detecta o formato automaticamente. Em ordem de preferência:

FormatoQuando usar
Multi-diaAba Workouts + abas <Dia> + <Dia> - Montagem
Grades por modalidadeAbas Individuais / Duplas + abas <Dia> + <Dia> - Montagem
Categoria × workout1 aba só com grade (categorias nas colunas, workouts nas linhas)
Template planoAbas Evento + WKT 1, WKT 2...

Aba Inscritos (opcional, recomendada)

Define faixas de número por categoria (ex: Scaled Feminino = 901–999, Scaled Masculino = 1001–1099). Usada pra desambiguar atletas em baterias mistas — quando duas categorias correm juntas, o sistema separa quem é de qual usando o número.

Aba Atleta(s) - X

Lista de atletas/duplas inscritas. Formato: número | nome | box, sem header. O sistema reconhece qualquer aba começando com "Atleta -" ou "Atletas -".

Como importar

  1. Botão ⚙ Configurar → aba Importar
  2. 📊 Importar Excel → seleciona o arquivo
  3. O sistema lê tudo e mostra resumo: N dias, N categorias, N atletas
  4. Se aparecer aviso, abra console (F12) pra ver detalhes — geralmente é número de atleta fora de qualquer faixa

Estrutura

Evento → Dias → Categorias → Workouts (+ baterias com atletas alocados).

Navegação

  • Dias aparecem como abas no header (só se tiver 2+)
  • Categorias no seletor à esquerda (mostra 1 por vez, com "1/N")
  • Workouts da categoria selecionada no painel principal

Tipos de workout

  • For Time — atleta completa tudo o mais rápido possível, com time cap
  • AMRAP — máximo de rounds em N minutos
  • Express — 2 fórmulas encadeadas (AMRAP + For Time)
  • For Load — N tentativas pra estabelecer carga máxima. Tem régua de anilhas marcáveis

Para criar/editar

  1. Passe o mouse sobre o workout pra ver botões de ação
  2. ✎ Editar abre o painel direito com todos os campos
  3. Pode mover entre dias/categorias no próprio editor
  4. Notas adicionais: campo livre que aparece acima da tabela. Use pro "Time cap: 10 min" ou regras especiais

IA (quando ativa)

Botão 💡 ao lado de Time Cap sugere um valor com base nos movimentos. Em AMRAP, n° de rounds é estimado automaticamente.

Baixar súmulas

No canto inferior esquerdo, o botão ⬇ Gerar súmulas tem 4 escopos:

Categoria selecionada(default) só a categoria do dropdown atual
Dia atualtodas as categorias do dia ativo
Evento inteirotodos os dias e categorias
Sem nomessúmulas em branco (sem atletas) — pra fotocópia

Estrutura do ZIP

Sábado/
  Scaled_Masculino/
    01_TWENTIES.html       ← todos os atletas dessa categoria/workout
    02_DOWNHILL.html
  Dupla_Misto/
    01_MAX_CLEAN.html      ← 1 página por dupla/time alocada

Em For Load com modalidade dupla/time, cada página representa o time inteiro (no modelo atual cada dupla é 1 entrada no Excel). A "MELHOR CARGA" da página é o resultado da equipe.

Imprimir

Abra o HTML no browser → Ctrl+P (Cmd+P no Mac) → "Salvar como PDF" ou imprimir direto. Cada atleta vira uma folha A4 separada (page-break-after automático).

Vários eventos ao mesmo tempo

O botão 📂 Eventos abre a lista de todos os eventos salvos no navegador. Você pode trabalhar em vários em paralelo sem perder dados.

Ações por evento

  • Abrir — troca pro evento clicado (o atual é salvo antes da troca)
  • ⎘ Duplicar — cria cópia com novo nome (útil pra ano seguinte)
  • ✎ Renomear — só muda o nome
  • 🗑 Arquivar — tira da lista principal mas não deleta (pode restaurar depois)

Arquivados

No fim do modal aparece ▶ Arquivados (N) se houver. Clique pra expandir. Cada arquivado tem 2 botões:

  • ↩ Restaurar — volta pra lista de ativos
  • ✕ Excluir definitivamente — remove do navegador (sem volta)

Eventos passados ficam aqui. Se for usar 1 vez por ano, arquivar é melhor que deletar.

Por que fazer backup?

Tudo fica salvo no navegador (localStorage). Se você limpar o cache, mudar de máquina, ou der pau no navegador, perde tudo. Backup em arquivo .json resolve isso.

Exportar

  1. ⚙ Configurar → aba Importar → 💾 Exportar JSON
  2. Baixa um arquivo nome-do-evento-data.json
  3. Se os logos somarem mais de 500KB, o sistema pergunta se quer incluir (pode gerar sem pra ficar mais leve)

Importar

  1. Mesma aba → 📂 Importar JSON
  2. Seleciona o arquivo .json salvo
  3. Sistema confirma se vai substituir o evento atual

Funciona pra mover entre máquinas, fazer backup antes de mudanças grandes, ou compartilhar com outra pessoa.

Apagar evento atual

Em ⚙ Configurar → aba Importar → botão vermelho Apagar evento. Limpa só o evento ativo, mantém os outros eventos salvos.

Reset completo

Apaga todos os eventos salvos, preferências e cache do navegador. Use só se quiser começar 100% do zero ou se algo bugar e quiser limpar tudo.

Não tem volta. Se tiver eventos importantes, exporte JSON antes (aba Backup).
